# Roblox Trends Parser & Tracker

Мощный парсер игр Roblox, предназначенный для отслеживания игровых трендов, анализа конкурентов и сбора статистики. Скрипт работает в фоновом режиме, собирает максимально подробные данные через официальные API Roblox и сохраняет их в Excel-таблицу.

## 🚀 Основные возможности

- **Автоматический мониторинг**: Работает 24/7, собирая данные каждые 30 минут.
- **Широкий охват**: Циклический обход категорий (Top, Trending, Simulator, Anime, Horror и др.) для поиска новых и перспективных игр.
- **Глубокая детализация**: Сбор более 30 параметров для каждой игры, включая:
    - Текущий онлайн (Playing)
    - Общее количество посещений (Visits)
    - Данные о разработчике (Creator)
    - Даты создания и последнего обновления
    - Жанры и технические характеристики
- **Умный экспорт в Excel**:
    - Автоматическое "выпрямление" (flattening) вложенных JSON-структур.
    - Добавление временных меток для каждой записи.
    - Сохранение истории (новые данные добавляются в конец файла).
- **REST API**: Встроенный сервер на FastAPI позволяет получать текущие тренды через веб-запросы.

## 🛠 Технологии

- **Python 3.8+**
- **FastAPI** & **Uvicorn**: Веб-сервер и API.
- **Pandas** & **OpenPyxl**: Обработка данных и работа с Excel.
- **Httpx**: Асинхронные HTTP-запросы к API Roblox.

## 📦 Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/ваш-логин/roblox-parser.git
   cd roblox-parser
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Запустите парсер:**
   ```bash
   uvicorn main:app --reload
   ```

После запуска скрипт начнет работу. Первый файл `roblox_trends.xlsx` появится в директории проекта через несколько секунд после старта.

## 📊 Структура данных (Excel)

Парсер сохраняет следующие ключевые поля:
- `fetch_timestamp` — точное время сбора данных.
- `name` — название игры.
- `playing` — текущее количество игроков.
- `visits` — общее количество визитов.
- `creator.name` — имя разработчика или название группы.
- `created` / `updated` — даты жизненного цикла игры.
- `id` — Universe ID игры.
- ... и еще более 20 технических полей.

## ⚙️ Настройка

Вы можете изменить список ключевых слов для поиска или интервал обновления в файле `main.py`:

- **Интервал**: Найдите `await asyncio.sleep(1800)` (1800 секунд = 30 минут).
- **Категории**: Измените список `search_queries` в функции `background_worker`.
